# 项目实施方案

用 ImageNet 预训练的 ConvNeXt Tiny 做 40 类垃圾分类，训练时用 5-fold 交叉验证，推理时做 fold 集成 + 轻量 TTA，最终生成符合比赛要求的 `result.txt`。

## 第 0 步：先明确输入输出

### 已知数据

- 训练集目录：`data/train/0` 到 `data/train/39`
- 测试集目录：`data/test/`
- 测试集文件名清单：`data/testpath.txt`
- 类别映射：`data/garbage_dict.json`

### 最终输出

生成 `result.txt`，要求：

- 恰好 400 行
- 每行格式：`图片名 标签`
- 标签必须是 `0..39`
- 顺序必须严格按照 `data/testpath.txt`

## 第 1 步：先做数据审计

这一步不要训练，先确认数据没问题。

### 1.1 检查训练集目录

确认：

- `data/train/0..39` 共 40 个文件夹都存在
- 每个文件夹里都有图片
- 图片扩展名统一可读，比如 `.jpg`、`.png`、`.jpeg`

### 1.2 检查测试集

确认：

- `data/testpath.txt` 恰好 400 行
- 每一行图片名在 `data/test/` 中都存在
- 之后推理时，只认 `testpath.txt`，不要直接遍历 `data/test/`

### 1.3 检查标签映射

看一下 `data/garbage_dict.json`：

- key 是否是 `"0"` 到 `"39"`
- value 是否是中文类别名
- 训练和提交时都只用数字标签，中文名称主要用于调试和分析

## 第 2 步：构建训练样本索引

不要依赖旧的 `train.txt` / `val.txt` 形式，直接从目录扫描。

### 2.1 扫描规则

遍历：

- `data/train/0/*.jpg`
- `data/train/1/*.jpg`
- ...
- `data/train/39/*.jpg`

每张图记录两项：

- `image_path`
- `label`

例如：

```text
data/train/12/img_001.jpg -> 12
```

### 2.2 标签使用原则

直接使用文件夹名作为标签：

- 文件夹 `0` 就是标签 `0`
- 文件夹 `39` 就是标签 `39`

不要重新编码成别的顺序，否则很容易提交错标签。

## 第 3 步：做 5-fold 分层划分

这是正式方案的核心之一。

### 3.1 为什么用 StratifiedKFold

因为类别大概率不均衡，分层划分可以让每一折的类别比例更稳定。

### 3.2 推荐设置

- `n_splits = 5`
- `shuffle = True`
- `random_state = 42`

### 3.3 每一折得到什么

每一折会得到：

- `train_idx`
- `val_idx`

然后训练一个独立模型：

- `fold0`
- `fold1`
- `fold2`
- `fold3`
- `fold4`

最终推理时对 5 个 fold 的输出做平均。

## 第 4 步：定义图像预处理和增强

垃圾分类场景背景乱、角度乱、光照乱，所以增强很重要，但不要太重。

### 4.1 输入尺寸

推荐分两阶段：

第一阶段：

- `img_size = 224`

先把流程跑通，速度快，显存压力小。

第二阶段：

- `img_size = 256`

正式冲分时再升到 256。如果显存足够，后期可以试 320，但不是第一优先级。

### 4.2 训练增强

推荐这一组：

- `RandomResizedCrop(224 or 256, scale=(0.8, 1.0))`
- `RandomHorizontalFlip(p=0.5)`
- `RandomRotation(10~15)`
- `ColorJitter(brightness=0.2, contrast=0.2, saturation=0.2, hue=0.05)`
- `ToTensor()`
- `Normalize(mean=ImageNet, std=ImageNet)`
- `RandomErasing(p=0.25)`

这套已经够用了。

### 4.3 验证 / 测试增强

验证和测试要稳定：

- `Resize`
- `CenterCrop` 或直接 `Resize((img_size, img_size))`
- `ToTensor()`
- `Normalize(mean=ImageNet, std=ImageNet)`

## 第 5 步：定义模型

### 5.1 主模型

首选：

- `ConvNeXt Tiny`
- `pretrained=True`
- `num_classes=40`

### 5.2 为什么它适合

- 比传统 ResNet 更强
- 比大 transformer 更稳
- 对自然图像分类迁移效果好
- 对细粒度差异有不错表现

### 5.3 预训练来源

直接用：

- `ImageNet-1K` 预训练权重

不要第一版就上更复杂的自监督预训练或别的大模型。

## 第 6 步：处理类别不均衡

这一步很重要。

### 6.1 先统计每类样本数

遍历训练集后，得到每个类别的样本数量。

例如：

- 类 `0`：300 张
- 类 `1`：120 张
- 类 `2`：500 张
- ...

### 6.2 推荐做法

先用带类别权重的 `CrossEntropyLoss`。

权重计算可以用比较稳的方式，比如：

```text
weight_c = total_samples / (num_classes * class_count_c)
```

或者更柔和一点：

```text
sqrt inverse frequency
```

也就是不要让少数类权重过大。

### 6.3 不建议一开始就上 Focal Loss

第一版先：

- `Weighted CrossEntropy`

等你发现确实有大量难分类尾部类，再考虑 `Focal Loss`。

## 第 7 步：训练配置

### 7.1 优化器

- `AdamW`

### 7.2 学习率

起步：

- `lr = 3e-4`

如果 batch 特别小，可以降到：

- `1e-4` 或 `2e-4`

### 7.3 weight decay

- `1e-4`

### 7.4 学习率调度器

推荐：

- `CosineAnnealingLR`
- 前 3~5 个 epoch 做 warmup

### 7.5 epoch 数

调试阶段：

- `10~15 epochs`

正式训练：

- `20~25 epochs`

### 7.6 batch size

按显存选：

- 8GB 显存：16 或 24
- 12GB 显存：32
- 更大显存：48 或 64

### 7.7 混合精度

必须开：

- `AMP`

这样显存更省，训练更快。

## 第 8 步：训练循环怎么做

每个 fold 都做完整训练。

### 8.1 每轮训练记录这些指标

至少记录：

- `train_loss`
- `val_loss`
- `val_acc`

建议再记录：

- `macro_f1`

因为类别不均衡时，单看 accuracy 不够。

### 8.2 保存最优模型

每个 fold 保存一个最佳 checkpoint，例如：

- `checkpoints/convnext_tiny_fold0_best.pth`
- `checkpoints/convnext_tiny_fold1_best.pth`
- ...
- `checkpoints/convnext_tiny_fold4_best.pth`

### 8.3 保存标准

建议优先按：

- `val_acc` 最大

如果你发现类别不均衡严重，也可以按：

- `macro_f1` 最大

## 第 9 步：先跑一个小基线确认流程

不要一开始直接 5 折全部跑满。

### 9.1 先跑单折

先只跑：

- `fold0`
- `img_size=224`
- `10~15 epochs`

目的不是拿最终成绩，而是确认：

- 数据读入没问题
- 标签没错
- 模型输出是 40 维
- 验证准确率在正常上升
- checkpoint 能保存
- 推理能跑通

### 9.2 如果单折流程没问题，再上完整 5-fold

这样能避免你后面浪费很多算力。

## 第 10 步：做 5-fold 正式训练

当单折验证没问题后，开始正式训练：

- `fold0 ~ fold4`
- 每折 `20~25 epochs`
- 输入尺寸升到 `256`
- 使用加权 CE
- 保存每折 best model

### 10.1 你最终会得到

5 个模型权重文件。这 5 个模型就是后面集成推理的基础。

## 第 11 步：验证阶段分析错误

这一步很值钱。

### 11.1 看混淆矩阵

你要知道哪些类最容易混。

例如可能出现：

- 外观相近的塑料制品互相混淆
- 某些厨余类别互相混淆
- 某些可回收物和其他垃圾在背景复杂时混淆

### 11.2 看每类准确率

如果某几个类别特别差，后面有几个补救方向：

- 提高输入尺寸
- 调整增强
- 调整类别权重
- 加第二个模型做融合

## 第 12 步：测试阶段推理

### 12.1 推理输入来源

严格读取：

- `data/testpath.txt`

不要直接遍历 `data/test/`。

### 12.2 每张图的推理过程

对于 `testpath.txt` 中每个文件名：

1. 读入 `data/test/<image_name>`
2. 做测试变换
3. 分别送入 5 个 fold 模型
4. 得到 5 份 logits 或 probability
5. 做平均
6. argmax 得到最终类别

## 第 13 步：加轻量 TTA

推荐只做一个最稳的：

- 原图
- 水平翻转图

### 13.1 TTA 的做法

对每个 fold、每张图：

- 预测原图
- 预测水平翻转图
- 两者概率平均

然后再对 5 个 fold 平均。

### 13.2 为什么只做这个

收益稳定、实现简单、不会显著拖慢太多。

不建议一开始做：

- 多裁剪 TTA
- 太多旋转 TTA
- 大量测试增强组合

## 第 14 步：生成 result.txt

这是提交最关键的一步。

### 14.1 写文件规则

每行写：

```text
image_name label_id
```

例如：

```text
test1.jpg 29
```

### 14.2 注意事项

- 无表头
- 一行一个结果
- 中间一个空格
- 标签必须是整数 `0..39`
- 行数必须正好 400
- 顺序必须和 `data/testpath.txt` 完全一致

## 第 15 步：提交前强校验

提交前一定做这几项检查。

### 15.1 行数检查

确认：

- `result.txt` 恰好 400 行

### 15.2 文件名检查

确认：

- `result.txt` 中的图片名顺序和 `data/testpath.txt` 完全一致

### 15.3 标签检查

确认：

- 所有标签都在 `0..39`

### 15.4 空行检查

确认：

- 没有空行
- 没有多余空格
- 没有最后一行异常格式

## 第 16 步：第一轮提分顺序

如果第一版能跑通，提分优先级按这个顺序来：

第一优先级：

- `224 -> 256`
- `1-fold -> 5-fold`

第二优先级：

- 加 `Weighted CE`
- 加 `horizontal flip TTA`

第三优先级：

- 第二模型融合：`EfficientNetV2-S`

第四优先级：

- 试 `320` 输入
- 试 `Mixup / CutMix`

## 第 17 步：简单集成怎么做

你前面提到“简单集成”，最推荐的是这个。

### 17.1 主模型

- `ConvNeXt Tiny`

### 17.2 第二模型

- `EfficientNetV2-S`

### 17.3 集成方式

对每张测试图：

- `ConvNeXt Tiny` 输出概率
- `EfficientNetV2-S` 输出概率
- 两者加权平均

例如：

- `0.6 * ConvNeXt + 0.4 * EfficientNet`

如果没有验证出明显差距，直接：

- `0.5 + 0.5`

### 17.4 什么时候再做

不是第一步就做。应该在单模型 5-fold 跑稳之后再做。

## 第 18 步：推荐的实际执行顺序

这是最实用的一条路线。

### 第一阶段：跑通

1. 审计数据
2. 扫描训练样本
3. 做 stratified split
4. 用 `ConvNeXt Tiny + ImageNet pretrained + 224`
5. 单折训练 `10~15 epoch`
6. 跑通推理
7. 生成一版 `result.txt`

### 第二阶段：做正式方案

1. 上 5-fold
2. 输入改成 256
3. 加 `Weighted CE`
4. 每折保存最优模型
5. 测试时做 fold ensemble
6. 加 hflip TTA

### 第三阶段：冲分

1. 引入 `EfficientNetV2-S`
2. 做双模型融合
3. 视显存尝试 320
4. 重新生成最终 `result.txt`

## 最终推荐参数表

### 主模型

- `ConvNeXt Tiny`
- `ImageNet-1K pretrained`
- `num_classes = 40`

### 训练

- `img_size = 256`
- `optimizer = AdamW`
- `lr = 3e-4`
- `weight_decay = 1e-4`
- `scheduler = cosine + warmup`
- `epochs = 20~25`
- `loss = Weighted CrossEntropy`
- `AMP = True`

### 数据

- `StratifiedKFold(n_splits=5, shuffle=True, random_state=42)`

### 增强

- `RandomResizedCrop`
- `HorizontalFlip`
- `RandomRotation`
- `ColorJitter`
- `RandomErasing`

### 推理

- 5-fold 平均
- 水平翻转 TTA
- 按 `data/testpath.txt` 顺序输出

## 一句话落地版

先用 ConvNeXt Tiny 的 ImageNet 预训练权重跑通单折，再升级到 5-fold + Weighted CE + 256 输入 + hflip TTA，最后视情况加 EfficientNetV2-S 做轻量融合，这就是这个比赛题最稳的实施路线。
