# 项目结构说明

```text
waste_sorting/
  README.md
  requirements.txt
  .gitignore
  data/
    train/
    test/
    garbage_dict.json
    testpath.txt
  docs/
    项目实施方案.md
    项目结构说明.md
    GitHub发布检查.md
  outputs/
    checkpoints/
    logs/
    predictions/
  scripts/
  src/
  temp/
    dataset.py
    train.py
    infer.py
```

## 根目录

- `README.md`：项目入口说明，面向第一次打开项目的人。
- `requirements.txt`：Python 依赖列表。
- `.gitignore`：控制哪些本地文件不提交到 GitHub。

## data

保存本地数据集与类别映射。由于图片文件通常体积较大，`data/train/` 和 `data/test/` 默认不提交到 GitHub。

## src

预留为后续正式源码目录。当前为了不改动已有临时脚本，暂时保持为空。

## temp

保存当前已有脚本，暂不改动：

- `dataset.py`：数据集读取与图片预处理。
- `train.py`：模型训练、验证、保存权重。
- `infer.py`：加载模型并执行推理。

## scripts

保存辅助脚本，例如：

- 生成 `train.txt`、`val.txt`、`test.txt`
- 统计类别数量
- 清理系统残留文件
- 检查图片是否损坏

## outputs

保存运行产物：

- `outputs/checkpoints/`：模型权重。
- `outputs/logs/`：训练日志。
- `outputs/predictions/`：预测结果。

这些内容默认不提交到 GitHub，只保留目录占位文件。
