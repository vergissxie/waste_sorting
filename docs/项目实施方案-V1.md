# 项目实施方案 V1

V1 方案用于确认项目主路线可行，目标是尽快跑通 40 类垃圾分类的完整训练、推理和提交流程，并拿到第一版可提交结果。

## 方案目标

- 完成 40 类单标签垃圾分类
- 跑通训练、验证、推理和提交校验
- 生成符合比赛要求的 `result.txt`
- 为后续 V2 提分方案提供稳定基线

## 数据输入与输出

### 已知数据

- 训练集目录：`data/train/0` 到 `data/train/39`
- 测试集目录：`data/test/`
- 测试集文件名清单：`data/testpath.txt`
- 类别映射：`data/garbage_dict.json`

### 最终输出

生成 `result.txt`，要求：

- 恰好 400 行
- 每行格式：`图片名 标签`
- 标签必须是 `0..39`
- 顺序必须严格按照 `data/testpath.txt`

## V1 推荐路线

### 1. 数据审计

先确认：

- `data/train/0..39` 共 40 个文件夹存在
- 每个文件夹里都有图片
- `data/testpath.txt` 恰好 400 行
- `data/testpath.txt` 中每个文件都存在于 `data/test/`
- `data/garbage_dict.json` 的 key 为 `0..39`

### 2. 样本构建

直接从 `data/train/0..39` 扫描训练样本：

- `image_path`
- `label`

标签直接使用目录名对应的整数，不做额外重映射。

### 3. 数据划分

V1 先用较轻量的验证方式跑通流程：

- 单折验证或 1 个 fold smoke run
- 后续再扩展到 5-fold

如果直接上完整方案，也可以使用：

- `StratifiedKFold(n_splits=5, shuffle=True, random_state=42)`

### 4. 模型选择

V1 主模型：

- `ConvNeXt Tiny`
- `ImageNet-1K pretrained`
- `num_classes=40`

原因：

- 迁移学习效果稳定
- 比老的 ResNet50 更强
- 训练和部署难度适中

### 5. 图像预处理与增强

训练增强建议：

- `RandomResizedCrop`
- `RandomHorizontalFlip`
- `RandomRotation(10~15)`
- `ColorJitter`
- `ToTensor()`
- `Normalize(mean=ImageNet, std=ImageNet)`
- `RandomErasing(p=0.25)`

验证 / 测试增强建议：

- `Resize((img_size, img_size))`
- `ToTensor()`
- `Normalize(mean=ImageNet, std=ImageNet)`

### 6. 训练配置

V1 推荐参数：

- `img_size = 224` 起步，跑通后升到 `256`
- `optimizer = AdamW`
- `lr = 3e-4`
- `weight_decay = 1e-4`
- `scheduler = cosine + warmup`
- `warmup = 3~5`
- `epochs = 10~15` 用于 smoke run
- `epochs = 20~25` 用于正式训练
- `AMP = True`

### 7. 类别不均衡处理

先使用：

- `Weighted CrossEntropy`

权重建议使用温和方式，例如：

- `sqrt inverse frequency`

### 8. 推理与提交

推理时必须：

- 严格按 `data/testpath.txt` 顺序读取测试图
- 输出 `image_name label_id`
- 生成 `result.txt`

V1 推理增强建议：

- fold ensemble
- 水平翻转 TTA

### 9. 提交校验

提交前检查：

- `result.txt` 恰好 400 行
- 文件名顺序与 `data/testpath.txt` 完全一致
- 标签范围在 `0..39`
- 无空行、无表头、无多余空格

## V1 目标结果

V1 的重点不是把分数拉到最高，而是获得一版：

- 训练可复现
- 推理可复现
- 提交格式稳定
- 可以作为后续 V2 实验对照基线

## 与 V2 的关系

- V1 负责验证主路线可行
- V2 负责将实现沉淀到 `src/`，并继续做提分实验

V2 方案见 [docs/项目实施方案-V2.md](项目实施方案-V2.md)
