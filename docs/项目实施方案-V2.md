# 项目实施方案 V2

V2 是当前正式发布方案，所有稳定实现均位于 `src/`。本版已经从单模型基线升级为双模型融合主线，并以 `92.5` 作为当前锁定发布成绩。

## 当前发布基线

- 平台最佳分数：`92.5`
- 对应结果文件：`result_e9_c6_e4_256_288.txt`
- 主模型：`ConvNeXt Tiny`
- 第二模型：`EfficientNetV2-S`
- 训练方式：`5-fold`
- 输入尺寸：`256`
- 推理策略：水平翻转 TTA + 多尺度 `256,288`
- 融合权重：`0.6 / 0.4`

## 代码对应关系

- `src/config.py`：路径、超参数和默认开关
- `src/dataset.py`：数据扫描、数据审计、训练增强、验证/测试变换
- `src/model.py`：`ConvNeXt Tiny` 和 `EfficientNetV2-S` 模型构建
- `src/train.py`：5-fold 训练、Weighted CE、warmup + cosine、AMP、best checkpoint 保存
- `src/infer.py`：单模型多 fold 推理，支持水平翻转 TTA 和多尺度推理
- `src/ensemble_infer.py`：双模型融合推理
- `src/validate_submission.py`：提交文件格式校验

## 当前默认配置

- `NUM_CLASSES = 40`
- `FOLDS = 5`
- `MODEL_NAME = "convnext_tiny"`
- `IMG_SIZE = 256`
- `BATCH_SIZE = 16`
- `EPOCHS = 25`
- `LR = 3e-4`
- `WEIGHT_DECAY = 1e-4`
- `WARMUP_EPOCHS = 3`
- `USE_WEIGHTED_CE = True`
- `SAVE_METRIC = "blend"`
- `AMP = True`

## 数据与输出约束

训练集与测试集目录约定如下：

```text
data/
  train/
    0/
    1/
    ...
    39/
  test/
  garbage_dict.json
  testpath.txt
```

提交文件要求：

- 输出文件名通常为 `result.txt`
- 恰好 `400` 行
- 每行格式为 `image_name<TAB>label_id`
- 标签范围必须是 `0..39`
- 顺序必须严格与 `data/testpath.txt` 一致

## 训练与推理主线

训练 ConvNeXt Tiny：

```powershell
.\.venv\Scripts\python.exe src\train.py
```

训练 EfficientNetV2-S：

```powershell
.\.venv\Scripts\python.exe src\train.py --model-name efficientnet_v2_s --checkpoint-prefix efficientnet_v2_s
```

生成当前发布版融合结果：

```powershell
.\.venv\Scripts\python.exe src\ensemble_infer.py --member convnext_tiny:convnext_tiny:0.6 --member efficientnet_v2_s:efficientnet_v2_s:0.4 --tta-scales 256,288 --output result.txt
.\.venv\Scripts\python.exe src\validate_submission.py result.txt
```

## 已完成实验结论

### 已纳入主线

- 评估阶段中心裁剪方案
- best checkpoint 多标准保存与对照
- 多尺度推理
- 第二模型 `EfficientNetV2-S`
- 双模型融合 `0.6 / 0.4`

### 不纳入主线

- 第一轮伪标签实验 `pl1`
- 原因：平台分数 `91.75`，低于当前 `92.5` 融合基线

## 后续方向

当前发布版已经锁定，不再继续微调 `0.7/0.3`、`0.6/0.4`、`0.5/0.5` 这类小权重差异。若后续继续冲分，优先顺序建议为：

1. 难样本与高频混淆类别分析
2. 更强第二模型或更强互补分支
3. 更高输入尺寸或更有针对性的推理策略

## 一句话总结

V2 的正式发布版，就是基于 `src/` 的双模型 5-fold 实现，以 `ConvNeXt Tiny + EfficientNetV2-S`、`256` 输入、`256,288` 多尺度推理和 `0.6 / 0.4` 融合作为当前锁定主线。
