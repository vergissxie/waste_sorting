# 项目实施方案 V2

V2 是当前正式方案，已经落地到 `src/` 目录。它在 V1 的基础上进一步规范了代码结构、训练入口、推理入口和提交流程，并给出后续提分实验顺序。

## 当前 V2 代码对应关系

当前方案和代码的匹配关系如下：

- 路径与训练超参数配置：`src/config.py`
- 数据扫描、数据审计、训练/验证/测试变换：`src/dataset.py`
- ConvNeXt Tiny 模型定义：`src/model.py`
- 5-fold 训练、Weighted CE、label smoothing、Mixup/CutMix、EMA、warmup + cosine、AMP：`src/train.py`
- 多 fold 推理、hflip TTA、多尺度 TTA、生成 `result.txt`：`src/infer.py`
- 提交文件格式校验：`src/validate_submission.py`

## V2 当前默认配置

V2 当前默认配置与本方案保持一致：

- `NUM_CLASSES = 40`
- `FOLDS = 5`
- `MODEL_NAME = "convnext_tiny"`
- `IMG_SIZE = 256`
- `BATCH_SIZE = 16`
- `EPOCHS = 25`
- `LR = 3e-4`
- `WEIGHT_DECAY = 1e-4`
- `WARMUP_EPOCHS = 3`
- `USE_WEIGHTED_CE = True`
- `LABEL_SMOOTHING = 0.05`
- `MIXUP_ALPHA = 0.2`
- `CUTMIX_ALPHA = 1.0`
- `MIX_PROB = 0.5`
- `EMA = True`
- `SAVE_METRIC = "blend"`
- `AMP = True`

## 第 0 步：先明确输入输出

### 已知数据

- 训练集目录：`data/train/0` 到 `data/train/39`
- 测试集目录：`data/test/`
- 测试集文件名清单：`data/testpath.txt`
- 类别映射：`data/garbage_dict.json`

### 最终输出

生成 `result.txt`，要求：

- 恰好 400 行
- 每行格式：`图片名 标签`
- 标签必须是 `0..39`
- 顺序必须严格按照 `data/testpath.txt`

## 第 1 步：先做数据审计

这一步不要训练，先确认数据没问题。

### 1.1 检查训练集目录

确认：

- `data/train/0..39` 共 40 个文件夹都存在
- 每个文件夹里都有图片
- 图片扩展名统一可读，比如 `.jpg`、`.png`、`.jpeg`

### 1.2 检查测试集

确认：

- `data/testpath.txt` 恰好 400 行
- 每一行图片名在 `data/test/` 中都存在
- 之后推理时，只认 `testpath.txt`，不要直接遍历 `data/test/`

### 1.3 检查标签映射

看一下 `data/garbage_dict.json`：

- key 是否是 `"0"` 到 `"39"`
- value 是否是中文类别名
- 训练和提交时都只用数字标签，中文名称主要用于调试和分析

## 第 2 步：构建训练样本索引

不要依赖旧的 `train.txt` / `val.txt` 形式，直接从目录扫描。

### 2.1 扫描规则

遍历：

- `data/train/0/*.jpg`
- `data/train/1/*.jpg`
- ...
- `data/train/39/*.jpg`

每张图记录两项：

- `image_path`
- `label`

例如：

```text
data/train/12/img_001.jpg -> 12
```

### 2.2 标签使用原则

直接使用文件夹名作为标签：

- 文件夹 `0` 就是标签 `0`
- 文件夹 `39` 就是标签 `39`

不要重新编码成别的顺序，否则很容易提交错标签。

## 第 3 步：做 5-fold 分层划分

这是正式方案的核心之一。

### 3.1 为什么用 StratifiedKFold

因为类别大概率不均衡，分层划分可以让每一折的类别比例更稳定。

### 3.2 推荐设置

- `n_splits = 5`
- `shuffle = True`
- `random_state = 42`

### 3.3 每一折得到什么

每一折会得到：

- `train_idx`
- `val_idx`

然后训练一个独立模型：

- `fold0`
- `fold1`
- `fold2`
- `fold3`
- `fold4`

最终推理时对 5 个 fold 的输出做平均。

## 第 4 步：定义图像预处理和增强

垃圾分类场景背景乱、角度乱、光照乱，所以增强很重要，但不要太重。

### 4.1 输入尺寸

V2 当前主配置：

- `img_size = 256`

后续提分实验可尝试：

- `img_size = 320`

### 4.2 训练增强

V2 当前使用：

- `RandomResizedCrop(img_size, scale=(0.8, 1.0))`
- `RandomHorizontalFlip(p=0.5)`
- `RandomRotation(15)`
- `ColorJitter(brightness=0.2, contrast=0.2, saturation=0.2, hue=0.05)`
- `ToTensor()`
- `Normalize(mean=ImageNet, std=ImageNet)`
- `RandomErasing(p=0.25)`

### 4.3 验证 / 测试增强

V2 当前实现使用：

- `Resize(round(img_size * 1.14))`
- `CenterCrop(img_size)`
- `ToTensor()`
- `Normalize(mean=ImageNet, std=ImageNet)`

后续提分实验可重点对照：

- `Resize(int(img_size * 1.14)) + CenterCrop(img_size)`
- 多尺度推理输入，例如 `256 / 288 / 320`

## 第 5 步：定义模型

### 5.1 主模型

首选：

- `ConvNeXt Tiny`
- `pretrained=True`
- `num_classes=40`

### 5.2 为什么它适合

- 比传统 ResNet 更强
- 比大 transformer 更稳
- 对自然图像分类迁移效果好
- 对细粒度差异有不错表现

### 5.3 预训练来源

直接用：

- `ImageNet-1K` 预训练权重

## 第 6 步：处理类别不均衡

### 6.1 先统计每类样本数

遍历训练集后，得到每个类别的样本数量。

### 6.2 推荐做法

V2 当前使用：

- `Weighted CrossEntropy`

权重采用温和的 `sqrt inverse frequency` 思路，避免少数类权重过大。

### 6.3 对照实验

后续可以做：

- `Weighted CE = True`
- `Weighted CE = False`

对照，看是否真的带来收益。

## 第 7 步：训练配置

### 7.1 优化器

- `AdamW`

### 7.2 学习率

- `lr = 3e-4`

### 7.3 weight decay

- `1e-4`

### 7.4 学习率调度器

- `warmup + cosine`

### 7.5 epoch 数

- `epochs = 25`

后续实验可试：

- `30~35`

### 7.6 batch size

V2 当前：

- `batch_size = 16`

后续实验可试：

- `24`
- `32`

### 7.7 混合精度

- `AMP = True`

## 第 8 步：训练循环怎么做

### 8.1 每轮训练记录这些指标

- `train_loss`
- `val_loss`
- `val_acc`
- `macro_f1`

### 8.2 保存最优模型

每个 fold 保存一个最佳 checkpoint，例如：

- `outputs/checkpoints/convnext_tiny_fold0_best.pth`
- `outputs/checkpoints/convnext_tiny_fold1_best.pth`
- ...
- `outputs/checkpoints/convnext_tiny_fold4_best.pth`

### 8.3 保存标准

V2 当前实现默认按：

- `0.5 * val_acc + 0.5 * macro_f1` 的综合分数最大

后续优化实验建议重点对照：

- `val_acc` 最大
- `macro_f1` 最大

## 第 9 步：做 5-fold 正式训练

V2 当前主线就是：

- `fold0 ~ fold4`
- 每折 `20~25 epochs`
- 输入尺寸 `256`
- 使用加权 CE
- 保存每折 best model

## 第 10 步：测试阶段推理

### 10.1 推理输入来源

严格读取：

- `data/testpath.txt`

不要直接遍历 `data/test/`。

### 10.2 每张图的推理过程

对于 `testpath.txt` 中每个文件名：

1. 读入 `data/test/<image_name>`
2. 做测试变换
3. 分别送入 5 个 fold 模型
4. 得到 5 份 logits 或 probability
5. 做平均
6. argmax 得到最终类别

## 第 11 步：加轻量 TTA

V2 当前使用：

- 原图
- 水平翻转图
- 可选多尺度：`--tta-sizes 256,288,320`
- 可选双模型融合：`--checkpoint-prefixes convnext_tiny_v2,efficientnet_v2_s_v2 --ensemble-weights 0.6,0.4`

### 11.1 当前做法

对每个 fold、每张图：

- 预测原图
- 预测水平翻转图
- 两者概率平均

然后再对 5 个 fold 平均。

### 11.2 后续提分方向

- 多尺度 TTA：`256 / 288 / 320`
- 与第二模型融合后的集成推理

## 第 12 步：生成 result.txt

每行写：

```text
image_name<TAB>label_id
```

例如：

```text
test1.jpg<TAB>29
```

注意事项：

- 无表头
- 一行一个结果
- 中间一个 tab 字符
- 标签必须是整数 `0..39`
- 行数必须正好 400
- 顺序必须和 `data/testpath.txt` 完全一致

## 第 13 步：提交前强校验

提交前一定做这几项检查：

- `result.txt` 恰好 400 行
- 文件名顺序和 `data/testpath.txt` 完全一致
- 所有标签都在 `0..39`
- 没有空行、没有多余空格、没有末尾异常格式

## 第 14 步：V2 提分实验顺序

在 V2 主线已经稳定、平台分数约 91% 的前提下，推荐按以下顺序继续实验：

1. 优先优化验证/测试变换，比较 `Resize((256,256))` 与 `Resize(292) + CenterCrop(256)`
2. 对照 best checkpoint 保存标准：`val_acc`、`macro_f1`、综合分数
3. 启用 label smoothing、Mixup/CutMix、EMA，并使用综合分数保存 best
4. 在推理阶段引入多尺度 TTA：`256 / 288 / 320` + hflip
5. 视显存尝试更大 batch 或更长 warmup，例如 `batch_size=24`、`warmup=5`
6. 尝试更高输入尺寸 `320`
7. 增加第二模型 `EfficientNetV2-S` 并做加权融合，推荐起步权重 `0.6 * ConvNeXt + 0.4 * EfficientNet`
8. 在高置信样本上尝试伪标签增强

## 一句话落地版

V2 的核心是：基于 `src/` 目录的正式实现，使用 ConvNeXt Tiny 的 ImageNet 预训练权重完成 5-fold + Weighted CE + 256 输入 + hflip TTA 的稳定主线，再通过验证变换、best checkpoint 标准、多尺度 TTA、第二模型融合和伪标签逐步提分。
